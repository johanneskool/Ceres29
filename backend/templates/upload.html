File specifications:

The tool expects to receive a comma separated file, a .csv file with semicolons as delimiters (i.e. the way Microsoft Excel creates them). These may or may not contain an extra semicolon at the end of each line. The expected encoding for the file is UTF-8. If your file does not adhere to the specifications, it may not get processed properly or in some cases don't get processed at all. In these cases, please try to format your data in such a way that it does comply to the standad.
The first row (the table heading) of the file should contain all node names. All further rows should start with these same node names, and these should be in the same order (it is not allowed to omit a row with all zeroes).
